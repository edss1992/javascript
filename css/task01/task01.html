FEX 是百度Web 前端研发部的别名，是社区基础技术部中的前端团队。其中 FE是 Front End 的缩写，X象征着团队不仅关注前端技术，还更重视全端及全栈能力。 FEX 致力于针对百度的各产品线在 Web 复杂应用，全端应用，全端数据监控及评估和前端工程优化等方面提升开发效率及页面体验，是百度内部最具影响力的前端团队。 FEX 是百度最早的开源实践者，先后推出了 Tangram、UEditor、FIS、GMU、Chassis、KityMinder 等库和工具。 不仅仅是「工程师」，也是「合伙人」 不会有人事无巨细地告诉工程师具体工作怎么做，要工程师自身去思考，而不仅仅是「执行」。 没有 PM，需要工程师主动去发现并解决问题。 可以成为任意项目的 PM，提出工程师的改进意见，并推动相关人员解决。 工程师也可以参与部门规范及制度的建设，而不仅仅只是遵守。 所参与的所有项目都会开源 内部平台项目会在内部开源 其它项目则在GitHub上开发 他们连面试题都是在 GitHub 上开源的[1] 团队之间没有秘密 工程师可以查看到所有项目的代码和文档及进展情况 工程师可以参加所有项目的例会 工程师可以查看到其他人的季度目标和计划 用到的将不仅是前端技术，为了达到目标，需要什么方面的技术就去学习，工程师们可以： 为了更好地分析前端数据，自己写 Hadoop 脚本分析日志，并自己搭建内部主从 MySQL 来分表存储 为了更好地支持本地开发效率，自行开发 PHP/JSP 调试服务器，使得前端开发可以不依赖后端 为了更好地结合 Native 和 Web 的优点，自行开发 Android 应用 SDK，让 Web 应用也能有 Native 的性能 为了更细致地分析页面性能，修改浏览器内核，开发细节分析工具 为了提升整体效率，和后端合作，寻求前后端一体化的解决方案 …… 在这里技术不是问题，有顶尖的大牛帮你排除技术障碍 有机会使用各种最新的技术来解决问题，比如： 用 SVG 绘制公式及图表，目前已开源 kityminder 用 HTML5 File API 编写上传组件 webuploader 用 nodejs、mongodb 开发内部服务 鼓励使用新技术，后续还将进行更多的尝试 「百度知道」「变形金刚百度知道彩蛋」特型效果 「百度搜索」「龙年百度搜索彩蛋」特型效果 「百度搜索」「2012世界末日 百度搜索彩蛋」特型地震效果 「百度搜索」「钢铁侠3百度彩蛋」特型效果 「百度阿拉丁」「动态天气应用效果」 这些熟知的高难度交互应用都是他们所创。 技术产品编辑 目前 FEX 不仅有很好的交互技术与经验，还将技术转化为通用基础产品，协助产品线不仅大幅提高了性能，还大大降低了人员的成本： Tangram - 前端脚本库 UEditor - 富文本编辑器 FIS - 前端开发整体解决方案 KityMinder - 矢量 Web 在线脑图查看编辑器 KityGraph - 大数据矢量图像绘制解决方案 KityFormula - 矢量公式全端解决方案 Webuploader - HTML5 高效上传解决方案 CROSS - 移动跨端高性能整体组件解决方案（专利） Chassis - 移动精简版 MVC[1] GMU - 移动 Webapp 精简版组件 alog - 整体前端统计框架 ... 等等 大事记编辑 2007 年成立「Web 前端研发部」 2008 年开源了前端脚本库「Tangram」 2011 年开源了富文本编辑器「UEditor」 2012 年开源了移动 Webapp 精简版组件「GMU」 2012 年在国内前端D2论坛讲解「FIS前端解决方案」Topic 2013 年 Web前端研发部启用别名「FEX」 2013 年开发了整体前端统计框架 「alog」，解决了前端统计框架空白 2013 年开源了前端开发整体解决方案「FIS」 2013 年开发了移动 Webapp 适用的精简版 MVC 框架「Chassis」 2013 年开发了浏览器工具 「PageTimeline」，协助产品线发现前端性能和提供性能准入数据 2014 年开源了矢量 Web 在线脑图查看编辑器「KityMinder」 2014 年开启独立域名博客「FEX」 2014 年开源了HTML5 高效前端上传解决方案「Webuploader」 2014 年用于内部的前端数据点评系统「dp」应用于轻应用统计和准入当中
